2. Features und Anforderungen

TODO: Stellen Sie die aus Ihrer Sicht wichtigsten funktionalen Anforderungen (maximal 10) zusammen:

    • Was soll Ihre Anwendung leisten?
      • Die Anwendung soll die Steuerung eines Autos aus der Vogelperspektive ermöglichen. Dabei kann der Anwender das Auto mithilfe der Pfeiltasten auf der Straße bewegen. Auf diesen Spuren muss sich der Anwender auch bewegen, da ihm auf diesen langsamere Autos oder andere Hindernisse, wie beispielsweise Öllachen oder Baustellen entgegenkommen können und er diesen Ausweichen muss. Berührt der Anwender mit dem Auto eines der Objekte, dann wird das Spiel abgebrochen und er befindet sich wieder im Hauptmenü. Dort kann er unter anderem zwischen 3 Schwierigkeitsgraden unterscheiden. (Leicht, Mittel, Schwer) Je höher der Schwierigkeitsgrad, desto schneller kommen dem Anwender die Objekte entgegen. Drückt der Anwender die rechte oder die linke Pfeiltaste, so kann er das Auto frei nach rechts oder links bewegen. Wird die obere bzw. die untere Pfeiltaste gedrückt, so kann der Anwender mit seinem Auto beschleunigen oder abbremsen. Daten, welche dauerhaft gespeichert werden sollen, sind der Highscore, also der zurückgelegte Weg pro Spiel und der Name des Anwenders. Das Programm soll auf Windows laufen und es wird mit Java und HTML entwickelt.
